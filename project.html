Project Idea

Overview

A very ambitious project idea of mine is a cloud computer. The main idea behind it is that people will no longer be bound to using multiple devices for different tasks, for example playing games on a game console, doing uni work on a laptop and using professional editing programs on a powerful computer – they will be able to do all of those tasks on any device they wish to use in the moment and then, should they choose to, even continue the exact same task on a different device right where they left it from. They won’t be limited to using only the devices they own either, they could just sign in to their account on their friends laptop or a computer at an internet café and gain instant access to all their files, programs and games with all of their personalised settings applied as soon as they sign in.

Motivation

A ‘cloud computer’ will make a lot of the aspects of current computers much simpler for the user. At the moment, a lot of people own multiple computers, which aren’t very interconnected and carrying on with the work, that was started on a MacBook at uni, on a PC at home is not a very simple and fast task. In this example, it could even be impossible if an application used was a macOS specific one. And even if files, that were worked on, are copied onto a USB stick or are backed up in the cloud, program settings aren’t (unless a program has a sync feature), so changing anything around in the program, it’s layout for example, will not carry across to another device. A ‘cloud computer’ completely eradicates this issue, as the work done on a device is not simply being saved onto it and kept there, it’s all done through the cloud. Let me explain.

Description

The way this would work is such. A ‘cloud computer’ is made up of a couple interconnected systems: First, it’s servers operated by me. Some of them simply store user’s data – their documents, pictures, videos and so on. Others, will be running and streaming programs to user’s devices. Not every program a user decides to use will necessarily be streamed, Microsoft Office for example, is not a very power consuming suit, so it could run directly on a user’s device. A AAA game, however, could potentially be a bit too much for that device to handle, so it will actually be running on one of my servers and streamed to the user’s device. Same goes for macOS applications, they could in theory be lightweight enough to be able to comfortably run on the device, but said device might not have the capability to run a macOS app, so it will be streamed from a server running macOS. Console games can also be streamed using this service. And the second system is user’s devices. Ideally, those devices should be developed by me and running my software to interconnect them in the most efficient way. The way this device would work is, let’s say someone owns a laptop of my creation. It will be quipped with 4G or 5G connectivity, so it’s always connected to the internet and has access to my servers. Once the user signs in to his account, instead of loading his user-account from a drive inside the laptop, it will be loaded up directly from my servers, with the drive inside the laptop working as a caching drive, similar to SSDs that accelerate larger HDDs. All programs will be loaded up directly from my servers, eliminating the need to instal them, meaning they don’t take any space and that every program a user owns is straight away accessible to them. For example, should they choose to open Spotify, it will load up from the server and run on the laptop and then if they launch Final Cut Pro, it will get launched on one of my servers and will be streamed onto the laptop. If they then decide to continue editing on their PC, the stream can be simply opened up on that computer and they can continue the work on it without having to reimport any files. Same goes for games, the user could start playing a game on their phone on a train, with the game being streamed onto their phone, and then continue playing on their TV at home. If the user is planning to go somewhere they know won’t have any broadband coverage, they can choose which programs and files should be kept directly on the computer, so that they could still use the device even without internet. A ‘cloud computer’ would be a subscription service, to pay off the expenses of running the service.

Tools and Technologies

The idea of a ‘cloud computer’ is not exactly reinventing the wheel. There are already services that exist that do a lot of the same things, such as OneDrive and iCloud to store documents online, GeForce Now and Google Stadia to stream games. My service simply combines them into one product, but works similarly to them. There will need to be servers storing user’s data and app data, most likely running Linux; VPS servers to stream programs and games, which will run different operating systems, depending on the program’s need. Parsec is a good low-latency streaming program, that has an purchasable SDK, which the streaming function of my service could be based off.

Skills Required

As much as I would love to say that this is a project I will be able to complete with my current skillset and budget (watch me instantly lose marks for saying this) – it’s not. Ideally, a large initial investment would be necessary to purchase servers and professional programmers would be needed to write the software (as I am definitely not literate in programming languages enough to do this… yet). A scaled down version of this project is somewhat feasible to complete now though. With a single computer acting as a server and a laptop running Windows that will be connected to the computer to view user files and stream programs from it. The drive caching aspect of my idea, sadly, isn’t feasible at the moment, however everything else will be a good enough product to showcase the capabilities of such a system.

Outcome

If a scaled down version of the project was created, it would still successfully eliminate the need to own a powerful computer or a gaming console alongside a laptop or a home computer. Users will be able to access all of their data from any device and will have access to any program they want through streaming. I don’t believe loading programs directly off the server onto the laptop would be feasible right now, so programs will still need to be installed locally on said laptop. However, if the AppData folder is possible to be stored on the server, all program settings and pre-sets can be used on any device the user wishes to. Streaming mac apps, although possible, will probably work poorly at start without further development due to Parsec not being able to stream directly from a macOS device, however can be done using a capture card and streaming off a different device, same goes for consoles. It’s not the best way to do this, but is feasible.
